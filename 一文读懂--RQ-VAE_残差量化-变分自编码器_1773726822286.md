论文：Autoregressive Image Generation using Residual Quantization地址：https://arxiv\.org/pdf/2203\.01941残差量化变分自动编码器（RQ\-VAE）是一种多级向量量化器， 在残差上进行量化来生成码字元组（语义ID）。 通过更新量化码本和DNN编码器\-解码器参数来联合训练自动编码器。\#\#\#\#\#\#RQ\-VAE（Residual Quantized Variational Autoencoder，残差量化变分自编码器）是变分自编码器（VAE）的一个改进版本，旨在提高生成模型的性能和生成样本的质量。RQ\-VAE引入了残差量化的思想，以克服传统VAE的一些局限性，如生成质量不高和潜在空间不够离散的问题。\#\#\#\#\#\#生成式推荐本质上，就是模拟大模型训练，使得参数scaling up（扩大规模）的一种尝试。因此当下相当多的工作，都是在对齐llm里的范式。接下来，就介绍生成式推荐中的tokenizer。为什么要有tokenizer如果把推荐照搬到llm中，本质上就是用很长很长的序列，去模拟llm的next token训练。但是nlp中的llm词表，一般是10w的量级；而推荐系统的item，则是上亿级别。因此，很难用上亿级别的词表去训练一个推荐大模型。此时，需要利用特别的分词手段。RQ\-VAE和RQ\-Kmeans就是典型的两类分词手段。主要思想就是，一个item用多个token\_id表示。例如，服装 = \[255, 33, 276, 12\]。这样，只需要用4 \* 256 个embedding词表，就能表示 2564256^4 个item。推荐系统里的分词器同样需要训练，这个训练是独立于推荐模型的\(和llm里tokenizer一样，都是提前训练好的\)。因此，后文主要是tokenizer的训练过程，和推荐模型本身无关。我们接下来来详细、清晰地阐述一下推荐系统中RQ\-VAE（Residual\-Quantized Variational Autoencoder）的Encoder、Decoder和Codebook是如何协同训练的。 我们可以将整个过程想象成一个“高级翻译和压缩”系统。 原始信息: 一个用户的行为历史（例如，他看过的电影序列）。Encoder \(编码器/翻译官\): 读取用户的历史，并将其翻译成一个简洁的、连续的“思想向量”（Latent Vector）。Codebook \(码本/数字词典\): 一本特殊的词典，里面有很多标准的“概念词条”（Code Vectors）。我们的目标是用这本词典里有限的词条来尽可能精确地表达“思想向量”。RQ\-VAE的精髓在于，它不是只查一次词典，而是进行多层次、残差式的查询，就像这样： 先用一个词条大致概括思想（比如“喜欢科幻”）。再看“思想向量”和这个词条的差异（残差），用第二个词条来描述这个差异（比如“偏爱太空歌剧”）。再看剩余的差异，用第三个词条来描述（比如“特别是复古风格的”）。…以此类推。 这样，多个简单的“概念词条”组合起来，就能非常精确地表达一个复杂的“思想”。\#\#\#下面我们分步详解。1、Encoder \(编码器\) 的训练\#\#\#\#编码器将输入数据 \(x\) 映射到一个潜在表示 \(z\)，这个潜在表示通常是一个连续的向量。与VAE相似，编码器输出潜在变量的分布参数（均值 μ\\mu 和标准差 σ\\sigma ）： q\(z\|x\)=N\(z;μ\(x\),σ\(x\)\)q\(z\|x\) = \\mathcal\{N\}\(z; \\mu\(x\), \\sigma\(x\)\) 将输入的用户行为序列 x 转换为一个高维、连续的隐向量 z\_e\(x\) 。训练目标: Encoder需要学会生成一个“优质”的隐向量 z\_e\(x\) 。所谓“优质”，有两层含义：（1）这个向量必须容易被Codebook量化。也就是说，它应该离Codebook中的某些码向量（或它们的组合）很近。（2）这个向量在被Codebook量化后，仍然保留了足够多的原始信息，以便Decoder能够成功地重建输入。如何训练? Encoder的参数更新主要由两个Loss驱动：Reconstruction Loss\(重构损失\): 这是最主要的驱动力。数据经过Encoder \-> Codebook \-> Decoder后得到重建结果 x’ 。计算 x’ 和原始输入 x 之间的差异（例如，对于序列推荐，使用交叉熵损失）。这个损失的梯度会反向传播，一路传回给Encoder。但是，Codebook的查询操作（找最近的邻居）是不可导的，这里用到了一个关键技巧——直通估计器 \(Straight\-Through Estimator, STE\)。我们后面会详述这个技巧。简单说，STE让梯度可以“无视”量化操作，直接从Decoder传给Encoder，告诉Encoder：“你需要调整你的输出 z\_e\(x\) ，使得最终的重建结果更接近真实值。”Commitment Loss \(承诺损失\): 这个损失用来约束Encoder，强迫它生成的 z\_e\(x\) 不要离被选中的码向量 z\_q\(x\) 太远。其公式通常是 \|\|z\_e\(x\) \- sg\(z\_q\(x\)\)\|\|² \(其中 sg 是 stop\_gradient ，意味着 z\_q\(x\) 不接收梯度，只作为目标\)。这个损失促使Encoder“承诺”其输出会紧靠码本，防止其输出向量在空间中随意漂移。总结: Encoder的训练由「重建损失」和「承诺损失」共同指导，目标是生成一个既能被码本很好地表示，又能被解码器成功复原的隐向量。2、Codebook \(码表\) 的训练\#\#\#\#为了引入离散化，RQ\-VAE在潜在表示 \(z\) 上进行残差量化。具体步骤如下：初始量化：将 \(z\) 映射到一个离散的代码簿（Codebook）中。残差计算：计算初始量化后的残差（Residual），即 \(z\) 与其量化表示之间的差异。残差量化：将残差再次进行量化，并累积到先前的量化结果中。通过多次残差量化，可以得到一个更精确、更离散的潜在表示。\#\#\#\#\#Codebook是一组可学习的向量（码向量/Embeddings）。在RQ\-VAE中，我们有多组Codebook（一个层级一个）。训练目标: Codebook里的码向量需要不断调整自己的位置，让自己成为数据在隐空间中的“最佳代表点”或“聚类中心”。一个好的Codebook应该能高效地覆盖Encoder产生的所有 z\_e\(x\) 向量的分布。如何训练? Codebook的训练通常由 Codebook Loss \(或称Quantization Loss 量化损失\) 驱动。对于第一层Codebook，其损失是 \|\|sg\(z\_e\(x\)\) \- c\_1\|\|² ，其中 c\_1 是从第一层Codebook中选出的、离 z\_e\(x\) 最近的码向量。 sg 操作意味着梯度只更新 c\_1 ，把它“拉向” z\_e\(x\) 的方向。对于第二层Codebook，其输入是第一步的残差 r\_1 = z\_e\(x\) \- c\_1 。其损失是 \|\|sg\(r\_1\) \- c\_2\|\|² ，其中 c\_2 是离残差, r\_1 最近的码向量。梯度会把 c\_2 拉向 r\_1 。以此类推，每一层的Codebook都在学习如何最好地表示上一层传递下来的残差。另一种常见的更新方式是使用指数移动平均（EMA），这种方法不依赖于优化器和梯度，而是通过历史数据平滑地更新码向量，通常更稳定。 总结: Codebook的训练目标是让其内部的码向量移动到数据点（或残差）密集区域的中心，成为这些数据的“最佳代言人”。3、Decoder \(解码器\) 的训练将量化后的向量z\_q\(x\) （它是从各层Codebook中选出的码向量之和：z\_q\(x\) = c\_1 \+ c\_2 \+ … \+ c\_M）作为输入，然后尝试重建原始的用户行为序列 x 。训练目标: 学习从离散、标准化的“概念组合”。 z\_q\(x\) 映射回复杂、具体的原始信息 x 的能力。如何训练? Decoder的训练完全由 Reconstruction Loss \(重够损失\) 驱动。计算重建输出 x’ 与原始输入 x 的差异。这个损失的梯度会直接反向传播更新Decoder的所有参数。这个过程非常直接，因为从 z\_q\(x\) 到 x’ 的路径是完全可导的（通常是多层神经网络）。总结: Decoder的训练目标很纯粹——尽最大努力将码本提供的标准化信息还原成原始输入。核心难点与解决方案：直通估计器 \(STE\) 整个训练过程的最大障碍是量化操作的不可导性。当你从Codebook中选择“最近”的那个码向量时，这个选择（ argmin ）操作的梯度几乎处处为零，无法进行反向传播。 STE的巧妙之处在于“欺骗”梯度： 前向传播时 \(Forward Pass\): 正常进行量化。即，输入一个连续向量z\_e\(x\)，输出一个离它最近的离散码向量z\_q\(x\)。反向传播时 \(Backward Pass\): 假装量化步骤不存在！直接将传到z\_q\(x\)的梯度 原封不动地复制 给z\_e\(x\)。前向传播:x 通过 Encoder 得到连续隐向量 z\_e\(x\) 。进行残差量化 \(RQ\): i\. 在第1层Codebook中找到离 z\_e\(x\) 最近的码向量 c\_1 。计算残差 r\_1 = z\_e\(x\) \- c\_1 。 ii\. 在第2层Codebook中找到离 r\_1 最近的码向量 c\_2 。计算残差 r\_2 = r\_1 \- c\_2 。 iii\. … 重复M次，得到码向量序列 \(c\_1, c\_2, …, c\_M\)。（也是最后分词的结果） iv\. 最终的量化向量是它们的和 z\_q\(x\) = c\_1 \+ c\_2 \+ … \+ c\_M 。 c\. z\_q\(x\) 通过 Decoder 得到重建的序列 x’ 。计算损失:a\. 个向量的码本。编码器输出很多个特征向量，每个特征向量都需要被量化（替换）为码本中最接近的那个向量。码本坍塌​ 指的是：在训练过程中，码本中只有一小部分向量（比如 K 个中的几十个）被频繁使用，而其他大部分向量永远得不到使用，成了“僵尸向量”。为什么会这样？因为训练是一个动态的“博弈”过程：编码器​ 倾向于输出那些更容易被码本中某个现有向量很好匹配的特征。一旦某些码本向量被证明是“好用”的，编码器就会把各种不同的特征都往这几个向量上靠拢。而其他码本向量因为一开始可能匹配得稍差，就被编码器“抛弃”了。既然没有特征指向它们，它们也就没有机会通过梯度更新变得更好。这就形成了一个恶性循环，导致码本的表达能力大大降低。RQ\-VAE 的解决方案：分而治之，责任到人RQ\-VAE 通过其分层残差量化的架构，从根源上打破了上述恶性循环。其核心思想是：不让任何一个码本层承担全部的描述责任，而是让每一层只负责“弥补”前一层的不足。1\. 职责分离与强制使用这是最根本的原因。在 VQ\-VAE 中：一个码本要负责描述图像中所有粒度的信息，从粗糙的轮廓到精细的纹理。这任务太重了，编码器自然会找“捷径”，只使用少数“全能型”的码本向量。在 RQ\-VAE 中：第一层码本：只负责捕捉最粗糙、最全局的信息（比如物体的形状和主要色块）。它的任务很简单：给出一个大概的轮廓。第二层码本：它的职责不是重新描述整个图像，而是描述第一层量化后留下的“残差”，即第一层无法描述的、更精细一层的信\-息（比如物体的部件、更细致的颜色过渡）。如果第一层已经做得很好，第二层的输入（残差）就会很小，但它依然必须工作，因为它的职责就是量化这个残差。后续层：以此类推，每一层都只处理前一层留下的残差，添加更细微的细节。总而言之，RQ\-VAE 通过其残差量化的架构，将复杂的表示任务分解成一系列简单的子任务，并强制每一层的码本都必须处理前一层输出的残差。这种“责任到人”的机制，从根本上防止了编码器投机取巧，使得所有码本向量都能被充分训练和使用，从而高效地解决了码本坍塌问题。\#\#\#RQ\-VAE 损失函数详解：第二步：分解 RQ\-VAE 的损失函数RQ\-VAE 的总损失函数由两大核心部分组成：重建损失​ \-
